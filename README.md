## TIL - 객체지향 프로그래밍

### 객체지향 프로그래밍

객체지향 프로그래밍은 프로그램을 객체라는 단위로 나누어서 만드는 프로그래밍 방식이다.  
객체는 쉽게 말해 하나의 물건이나 캐릭터 같은 것이라고 생각하면 이해하기 쉽다.

예를 들어 게임을 만든다고 하면 플레이어, 몬스터, 아이템 같은 것들이 각각 하나의 객체가 될 수 있다.  
이 객체들은 자신만의 정보와 행동을 가지고 있다.

예를 들어 Player라는 객체가 있다면

체력  
공격력  
이동하기  
공격하기  

같은 것들이 있을 수 있다.

이처럼 관련된 데이터와 기능을 하나로 묶어서 프로그램을 만드는 것이 객체지향 프로그래밍이다.

### 클래스와 객체

클래스는 객체를 만들기 위한 설계도 같은 것이다.  
그리고 객체는 그 설계도를 이용해서 실제로 만들어진 것이다.

예를 들어 붕어빵 기계로 생각해보면:  
붕어빵 틀은 클래스를 의미하고, 그 틀로 만들어진 붕어빵은 객체라고 할 수 있다.
붕어빵 맛있겠다.
### 객체지향의 특징

# 1. 캡슐화  
데이터와 기능을 하나의 객체 안에 묶는 것이다.

# 2. 상속  
기존 클래스의 기능을 물려받아서 새로운 클래스를 만들 수 있다.

# 3. 다형성  
같은 기능이지만 상황에 따라 다르게 동작할 수 있다.

# 4. 추상화  
복잡한 내부 구조는 숨기고 필요한 기능만 보여주는 것이다.
