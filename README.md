## TIL - 붕어빵 객체지향 프로그래밍

### 객체지향 프로그래밍

객체지향 프로그래밍은 프로그램을 객체라는 단위로 나누어서 만드는 프로그래밍 방식이다.
객체는 쉽게 말해 하나의 물건 같은 것이라고 생각하면 이해하기 쉽다.

예를 들어 붕어빵이라는 객체가 있다면

반죽
팥 또는 슈크림
모양
굽기

같은 것들이 있을 수 있다.

이처럼 하나의 붕어빵 안에 여러 정보와 기능이 함께 들어 있는 것처럼,
객체지향 프로그래밍도 관련된 데이터와 기능을 하나로 묶어서 프로그램을 만드는 방식이다.

### 클래스와 객체

클래스는 객체를 만들기 위한 설계도 같은 것이다.  
그리고 객체는 그 설계도를 이용해서 실제로 만들어진 것이다.

예를 들어 붕어빵 기계로 생각해보면
붕어빵 틀은 클래스를 의미하고, 그 틀로 만들어진 붕어빵은 객체라고 할 수 있다.
붕어빵 맛있겠다.

### 객체지향의 특징

객체지향의 특징도 붕어빵으로 생각할 수 있다.

## 캡슐화  
피자 붕어빵 안에는 반죽뿐만이 아니라 토마토소스,치즈 등이 들어가있다.
이처럼 하나의 붕어빵 안에 여러 요소가 함께 들어 있는 것처럼, 객체지향에서도 데이터와 기능을 하나의 객체 안에 묶어 놓는다.
피자 붕어빵을 별로 못먹어봐서 먹어보고 싶다.

## 상속  
붕어빵 틀을 기본으로 해서 팥 붕어빵, 슈크림 붕어빵, 치즈 붕어빵 같은 여러 종류를 만들 수 있다.  
이처럼 기존의 클래스(붕어빵 틀)를 기반으로 새로운 클래스(신메뉴 붕어빵)를 만들어 기능을 이어받는 것을 상속이라고 한다.

## 다형성  
같은 붕어빵이라도 안에 들어 있는 내용물(팥 또는 슈크림 등)에 따라 맛이 달라진다.  
이처럼 같은 기능이라도 상황이나 객체에 따라 다르게 동작할 수 있는 것을 다형성이라고 한다.

## 추상화  
대부분의 사람들은 붕어빵을 먹을 때 안에서 어떤 과정으로 만들어졌는지까지는 신경 쓰지 않는다.  
그냥 붕어빵이라는 결과만 보고 먹는다.  
이처럼 복잡한 내부 구조는 숨기고 중요한 부분만 보여주는 것을 추상화라고 한다. 난 과정이 살짝 궁금한 편이다.
